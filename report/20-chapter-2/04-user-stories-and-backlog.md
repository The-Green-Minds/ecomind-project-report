## Requirements Specification

Esta sección conserva la estructura exigida para AV1, pero no convierte assumptions en requisitos confirmados. Su contenido se completará después de codificar las entrevistas, producir los artefactos de Needfinding y acordar el lenguaje ubicuo.

### User Stories

**Estado:** pendiente de entrevistas. Cada User Story deberá vincular persona o rol validado, evidencia, pain point, regla de negocio, criterios de aceptación y hypothesis. No se aceptarán historias sin trazabilidad ni criterios verificables.

| ID | User Story | Segmento / fuente | Aceptación | Hypothesis | Estado |
|---|---|---|---|---|---|
| US-XX | Pendiente | Participante y video | G/W/T pendiente | Pendiente | No redactada |

### Impact Mapping

**Herramienta:** UXPressia. El artefacto deberá incluir goal medible, actors, impacts y deliverables; cada rama se enlazará con entrevistas y con las User Stories. Se incorporarán la URL editable y la exportación. **Estado:** pendiente de Needfinding.

### Product Backlog

**Herramienta adoptada:** Jira. El backlog contendrá épicas, User Stories, tareas técnicas, criterios de aceptación, prioridad, estimación, dependencias y hito. El informe incluirá enlace y capturas legibles; GitHub conservará únicamente el resumen trazable.
